Teoria y Práctica de GIT y GITHUG

Git es una herramienta que nos ayuda a controlar el avance que va teniendo nuestro proyecto con respecto al tiempo

Áreas de Git:

Son en estas áreas que se van moviendo los cambios o avances que se tiene del proyecto a medida que va creciendo:
Working Directory
Staging area
Repository git

Comandos de GIT

git commit: se abre un editor en la terminal para escribir lo agregado o cambiado desde Working directory al Staging area con el “git add”
git add . : el punto se refiere a que agregará a todos los archivos que han sido creados o sufrido algún cambio en el Working directory.
git commit -m “texto alusivo a lo agregado” : de esta manera se evita abrir el editor en la terminal.
git log : Se pueden apreciar los commits (Cambios) que se lleva en el proyecto. (Entro en un estado de archivos, para salir uso la tecla Q).
git status : Nos muestra los archivos que están agregados (verde) al Staging area y los que faltan(rojo) por agregar. Para luego ser commiteados.
rm+ nombre_archivo: nos permite eliminar un archivo seleccionado del repositorio, sin subir los cambios al staging.
git rm + nombre_archivo: Nos permite eliminar un archivo y adicional agrega los cambios al staging.
git restore - - staged + nombre_archivo: nos permite sacar algún cambio subidos al staging. siempre y cuando el archivo no haya sido commiteado.
mv + nombre_archivo + nuevo_nombreArchivo: Permite mover o renombrar archivos, sin subir los cambios al staging.
git mv + nombre_archivo + nuevo_nombreArchivo: Permite mover o renombrar archivos y adicional agrega los cambios al staging.
Ignorando archivos y directorios: Esto nos permite ignorar archivos para que no sean incluidos nunca en los repositorios de git. Esto se hace por la necesidad de crear archivos de configuración se sean específicos para nuestra máquina. Ejemplo: variables de entornos (.env).
Se crea un archivo en el editor de código llamado .env
Se crea un archivo en el editor de código llamado .gitignore
Dentro de .gitignore se introduce .env.
Al hacer git status solo nos tocará agregar .gitignore al repositorio.
git status -s: Permite ver cambios de una manera más visual.
git log —oneline: Permite revisar el historial de todo nuestro proyecto.
git branch: Nos muestra la rama en que se esta trabajando.
git branch + nombre-rama/ git checkout + nombre-rama: Nos crea una nueva rama de trabajo.
git checkout -b “features/nombre-de-la-funcionalidad”
git checkout -b “SS-4515 (Nombre del tickets)”
git checkout -b ramab : se crea una rama nueva de trabajo de manera más rápida.
git checkout + nombre-rama: Nos lleva a la rama mencionada.
cat + nombre-archivo: Nos muestra el contenido de nuestros archivos.
git merge + nombre-rama: Nos trae toda la rama especificada a la rama actual ubicada.
git remote add origin https://github.com/hug0f/miweb.git :
git remote: Nos sirve para indicar si vamos a tener un servidor remoto en el cual vamos a subir nuestro cambios.
add origin: les estamos indicando de donde vamos a buscar nuestro cambios y a donde tenemos que subir nuestros cambios.
url: dónde se encuentra almacenado nuestro código.
git push -u origin main:
git push: Es con el cual podemos subir nuestros cambios, los cuales serán con respecto a la rama que se este trabajando.

Algunos Comandos dentro de la terminal:

ls: Nos permite listar todos los archivos y carpetas que se encuentran en el directorio.
pwd: nos permite ubicarnos en la carpeta donde nos encontremos.
cd: nos permite movernos entre las diferentes carpetas enlistadas. (Ejemplo: cd nombre_carpeta/). cd .. : nos permite regresar a la carpeta anterior.
mkdir + nombre_nueva_carpeta: nos permite crear una nueva carpeta en el sitio que nos encontremos ubicados.
command + k: limpio la terminal.
